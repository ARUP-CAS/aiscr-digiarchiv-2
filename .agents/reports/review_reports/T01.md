## 1. Shrnutí zjištění

Repozitář obsahuje produkční webovou aplikaci **Digitální archiv AMČR** — prezentační vrstvu
pro archeologická data AIS CR. Architektura je tvořena čtyřmi hlavními komponentami:

1. **Java/Servlet backend** (`web/`) — 88 Java tříd organizovaných do 5 balíčků. Aplikace
   komunikuje s Fedora Repository REST API pro čtení dat a s Apache Solr pro vyhledávání.
   Packaging je WAR (nasazení na servlet kontejner, pravděpodobně Tomcat).

2. **Angular frontend** (`web/src/main/ng/`) — TypeScript/Angular SPA sestavená Maven pluginem
   jako součást WAR archivu. Obsahuje komponenty, stránky, sdílené utility a SCSS styly.

3. **Apache Solr** (`solr/`) — 14 kolekcí pokrývajících různé typy dat. Vyhledávání je klíčovou
   funkcí systému; každý typ AMČR záznamu má vlastní Searcher třídu.

4. **ThumbnailsGenerator** (`ThumbnailsGenerator/`) — samostatný Maven modul pro dávkové
   generování miniatur obrázků a PDF z Fedora Repository a jejich indexaci do Solr.

Systém implementuje OAI-PMH protokol pro sklizeň metadat a podporuje více verzí AMČR API
(v2.0, v2.1, v2.2) prostřednictvím XSLT transformací.

## 2. Identifikované problémy

### P01 — Absence automatizovaných testů
- **Zjištění:** Adresář `web/src/test` neexistuje. Projekt nemá žádné automatizované Java testy.
- **Dopad:** Regresní změny nemohou být automaticky odhaleny; CI pipeline nemůže ověřit
  správnost kódu před nasazením.
- **Závažnost:** Vysoká

### P02 — Překlep v názvu Java balíčku
- **Zjištění:** Balíček `cz.inovatika.arup.digiarchiv.web4.imagging` obsahuje překlep
  (`imagging` místo `imaging`).
- **Dopad:** Překlep se prolíná importy a je viditelný v dokumentaci. Oprava vyžaduje
  přejmenování balíčku a všech importů.
- **Závažnost:** Nízká (kosmetická, ale matoucí)

### P03 — Nekonzistentní base package v ThumbnailsGenerator
- **Zjištění:** ThumbnailsGenerator používá `cz.incad.arup.thumbnailsgenerator`, zatímco
  hlavní aplikace používá `cz.inovatika.arup.digiarchiv.web4`. Jiná organizace (`incad` vs
  `inovatika`) naznačuje historicky odlišný původ modulu.
- **Dopad:** Ztěžuje orientaci v kódové základně a vlastnictví kódu.
- **Závažnost:** Nízká

### P04 — Chybějící managed-schema pro kolekci `oai`
- **Zjištění:** Kolekce `solr/oai/conf/` neobsahuje soubor `managed-schema`, který mají
  všechny ostatní kolekce.
- **Dopad:** Nejasné, zda kolekce používá výchozí schéma nebo dynamická pole. Riziko
  neočekávaného chování při indexaci.
- **Závažnost:** Střední — bude podrobněji prošetřeno v T03.

### P05 — Chybějící Docker/infrastrukturní konfigurace
- **Zjištění:** V repozitáři nejsou žádné Dockerfile ani docker-compose soubory.
  Infrastrukturní konfigurace je pravděpodobně v externím repozitáři nebo v `.gitignore`.
- **Dopad:** Není možné lokálně spustit aplikaci z tohoto repozitáře bez dalších zdrojů.
  Komplikuje onboarding nových vývojářů.
- **Závažnost:** Střední

### P06 — 14 Solr kolekcí bez zjevné konsolidační strategie
- **Zjištění:** Systém používá 14 samostatných Solr kolekcí. 10 z nich sdílí identický
  `solrconfig.xml`, což naznačuje, že byl zkopírován bez přizpůsobení.
- **Dopad:** Správa a aktualizace konfigurace je obtížnější; změna v `solrconfig.xml` musí
  být aplikována na 10 souborů.
- **Závažnost:** Střední

## 3. Návrhy zlepšení

1. **Testy:** Přidat alespoň základní unit testy pro klíčové třídy (SearchUtils, IndexUtils,
   FormatUtils, FedoraUtils). Zvážit integrační testy pro Solr dotazy.

2. **Překlep `imagging`:** Přejmenovat balíček na `imaging`. Lze provést automaticky v IDE
   (refactor → rename package) s aktualizací všech importů.

3. **Solr `solrconfig.xml`:** Zvážit použití Solr configset inheritance nebo udržovat sdílený
   `solrconfig.xml` na jednom místě a symlinky/sdílení přes CI.

4. **Docker konfigurace:** Dokumentovat v README, kde se nachází infrastrukturní konfigurace
   (docker-compose), nebo zvážit přidání vývojového `docker-compose.yml` přímo do repozitáře.

5. **`oai` kolekce:** Ověřit, zda absence `managed-schema` je záměrná (dynamické schéma)
   nebo chyba. Přidat komentář do `solrconfig.xml`.
